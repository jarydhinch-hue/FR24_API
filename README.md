# FR24_API

The code file has plenty of markup throughout that explains each line's purpose and what to include. The code assumes you are following a similar framework
as our project (pulling flight entries based on a list of registration IDs), but can be adapted for other references, like all flights in a certain geographic boundary, etc. FR24's official website has more details about getting started: https://fr24api.flightradar24.com/docs/getting-started

A side note; each plan level for credits has a different date range allowed for pulling, so depending on your study period, you may need to elevate your choice from the $9 to the $90 plan, even if you do not need that many credits. The $9 plan extends back 30 days from the present moment, while the $90 plan extends back 2 years. Note this is likely subject to change as FR24 tweaks their system, but at the time of writing, this is the ruleset.
